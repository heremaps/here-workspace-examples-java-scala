  20.  def listSchemas(access: Option[String] = None, sort: Option[String] = None, order: Option[String] = None, from: Option[String] = None, groupId: Option[String] = None, artifactId: Option[String] = None, limit: Option[Int] = None, offset: Option[Long] = None, type: Option[String] = None, linkable: Option[Boolean] = None, layerSchema: Option[Boolean] = None): ApiRequest[ListSchemasResponse]

    Return the list of available schemas.

    Return the list of available schemas. If scoped token is used, then the list of schemas is returned within the project the token belongs to. If unscoped token is used, then the list of schemas is returned including the schemas belonging to the projects.

    Path: /schema

    Method: GET

    Expected answers:

    code 200 : ListSchemasResponse (OK)

    code 400 : (Invalid schema list parameters were provided)

    code 401 : HapiCompliantSystemError (You are not authorized to view the schema or have wrong / invalid credentials)

    code 403 : ErrorResponse (Accessing the schema is forbidden)

    code 500 : ErrorResponse (Internal server error. Support ticket should be created)

    access

    Comma separated list of access types (with OR semantic) to filter result.If 'default', returns schema user have read access to. If 'orgAdmin',returns catalogs OrgAdmin user have access to.

    sort

    Applies sorting based on specified fields. Allowed values are group, name, version. Multiple fields must be separated with ':'

    order

    Applies ordering. Defaults is ASC

    from

    Applies 'greater than' filtering to fields specified in 'sort' property. For example when 'sort' property is 'name:version' and 'from' is 'b:1.0.0' then the response will contain records with a name starting from 'b' till 'z' and version greater than 1.0.0

    groupId

    Applies filtering based on groupId

    artifactId

    Applies filtering based on artifactId

    limit

    Specifies the maximum amount of records that could be returned. Default is 20

    offset

    Specifies how many records will be skipped at the beginning. Default is 0

    linkable

    Indicates if the response should only return the list of linkable schemas. Note: - If set to 'true' and the access token has a project scope, then the response will include the schemas that are available to the caller to link to the project in the token's project scope and HERE public schemas. - If set to 'true' and the access token has no scope, then the response will include all HERE public schema only.\"

    layerSchema

    Indicates if the response should only return the list of layer schemas. If value set to false the response will also include non-layer schemas
