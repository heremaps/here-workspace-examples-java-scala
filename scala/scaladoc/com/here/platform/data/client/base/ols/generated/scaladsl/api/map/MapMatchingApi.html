  14.  def matchedPath(baseUrl: Option[String] = None, version: String, roadAccess: String, trace: Trace, interpolate: Option[Boolean] = None)(implicit traceBodySerializer: BodySerializer[Trace]): ApiRequest[MatchedPath]

    Snaps a sequence of geocoordinates to the specified map version and gets information about the geometry of the roads and their identifiers.

    Snaps a sequence of geocoordinates to the specified map version and gets information about the geometry of the roads and their identifiers. For best results, use 1Hz GPS data or any points that have a spacing of a few meters between them. In some situations, when consecutive points are too far apart (more than about 30 kilometers of on-road distance), they could be considered unreachable and this could result in a matching error. Please note that u-turns are not explicitly supported and that the road access for the vehicle modes (e.g. car) does not support time dependent or complex restrictions.

    Path: /maps/versions/{version}/matchedPath

    Method: POST

    Expected answers:

    code 200 : MatchedPath (The matched path corresponding to the input trace)

    code 400 : ErrorResponse (Input data failed validation)

    code 404 : ErrorResponse (Map version not found)

    baseUrl

    Base URL without path parameters and query.

    version

    Map version to use, or latest to use the latest available version.

    roadAccess

    The map matching mode to use. If unrestricted, the matched path will match all roads and directions without taking restrictions into consideration. When car is specified, the driving direction of roads and forbidden turns are considered and discarded. Note that the car mode does not take complex or time-dependent driving restrictions into consideration.

    trace

    The trace object to be matched. Note that there is a lower and an upper limit on the number of points in the trace.

    interpolate

    Specifies whether to add interpolated points to subpaths in order to retrieve the full path geometry. When interpolate=true, additional interpolated points will be inserted between onRoad ones, leading to subpaths that will follow the road geometry in a smoother way. Since this involves more computation and, therefore, impacts response times, the default value is false.
