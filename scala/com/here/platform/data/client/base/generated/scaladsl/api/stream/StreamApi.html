  6.  def commitOffsets(hrn: String, baseUrl: Option[String] = None, layerId: String, commitOffsetsRequest: CommitOffsetsRequest, subscriptionId: Option[String] = None, mode: Option[String] = None)(implicit commitOffsetsRequestBodySerializer: BodySerializer[CommitOffsetsRequest]): ApiRequest[Unit]

    After reading data, you should commit the offset of the last message read from each partition so that your application can resume reading new messages from the correct partition in the event that there is a disruption to the subscription, such as an application crash.

    After reading data, you should commit the offset of the last message read from each partition so that your application can resume reading new messages from the correct partition in the event that there is a disruption to the subscription, such as an application crash. An offset can also be useful if you delete a subscription then recreate a subscription for the same layer, because the new subscription can start reading data from the offset. To read messages already committed, use the /seek endpoint, then use /partitions. The base path to use is the value of 'nodeBaseURL' returned from /subscribe POST request.

    Path: /layers/{layerId}/offsets

    Method: PUT

    Expected answers:

    code 200 : (OK, offsets committed) Headers : X-Correlation-ID -

    code 202 : (OK, offsets committed. This status indicates that your multi-region catalog had a failover or failback event, and you have committed offsets from previous region. No actions are needed from you.) Headers : X-Correlation-ID -

    code 400 : ErrorResponse (Bad Request)

    code 401 : InlineResponse401 (401 Unauthorized)

    code 403 : InlineResponse403 (403 Forbidden)

    code 404 : ErrorResponse (Not found)

    code 409 : ErrorResponse (Conflict)

    code 412 : ErrorResponse (Pre-condition failed)

    code 500 : ErrorResponse (Internal Server Error)

    hrn

    The HERE Resource Name (HRN) of the catalog.

    baseUrl

    Base URL without path parameters and query.

    layerId

    The ID of the stream layers.

    commitOffsetsRequest

    The offsets to commit. It should be same as the offset of the message you wish to commit. Do not pass offset + 1 as mentioned in Kafka Consumer documentation. The service adds one to the offset you specify.

    subscriptionId

    The subscriptionId received in the response of the /subscribe request (required if mode=parallel).

    mode

    The subscription mode of this subscriptionId (as provided in /subscribe POST API).

    Consumes data from a layer.     Enables message consumption from a specific stream layer (topic).

    Enables message consumption from a specific stream layer (topic). Use the base path returned from the API Lookup service. The number of subscriptions within the same group cannot exceed the parallelism allowed. For more details see [Get Data from a Stream Layer](https://developer.here.com/olp/documentation/data-store/data_dev_guide/rest/getting-data-stream.html).

    Path: /layers/{layerId}/subscribe

    Method: POST

    Expected answers:

    code 201 : ConsumerSubscribeResponse (OK) Headers : X-Correlation-ID -

    code 400 : ErrorResponse (Bad Request)

    code 401 : InlineResponse401 (401 Unauthorized)

    code 403 : InlineResponse403 (403 Forbidden)

    code 404 : ErrorResponse (Not found)

    code 412 : ErrorResponse (Pre-condition failed)

    code 500 : ErrorResponse (Internal Server Error)

    code 503 : ErrorResponse (Service Unavailable error)

    hrn

    The HERE Resource Name (HRN) of the catalog.

    layerId

    The ID of the stream layers.

    mode

    Specifies whether to use serial or parallel subscription mode. For more details see [Get Data from a Stream Layer](https://developer.here.com/olp/documentation/data-store/data_dev_guide/rest/getting-data-stream.html).

    subscriptionId

    Include this parameter if you want to look up the nodeBaseURL for a given subscriptionId.
